= Failure Triage Guide
:toc:

== Purpose
Provide deterministic triage steps for common operational failures.

== Triage Matrix
[cols="2,2,3",options="header"]
|===
| Symptom | Signal | Triage steps

| Metadata fallback on startup
| `metadata.load.fallback`
| Check API reachability and timeout settings, then retry to confirm fallback clears.

| Catalog load timeout/failure
| `catalog.load.timeout` or `catalog.load.failure`
| Verify network/API endpoint, inspect cause fields, retry with `Ctrl+R` in TUI or rerun headless.

| Validation failure
| `cli.validation.failed` or validation error output
| Correct invalid field values (group/artifact/package/output/extension) and rerun.

| Archive/generation failure
| `generate.execute.failure` or `generate.execute.timeout`
| Check output-directory permissions/conflicts, API health, and archive safety messages.

| Unexpected TUI termination
| `tui.session.failure`
| Re-run with `--verbose`, capture traceId and event sequence, confirm backend selection policy.
|===

== Operator Checklist
. Reproduce with `--verbose` and capture `stderr` diagnostics lines.
. Group events by `traceId` for one execution attempt.
. Classify by phase: startup, catalog load, generation, post-actions.
. Apply symptom-specific matrix steps and rerun.
. If persistent, attach diagnostics excerpt and command used to incident report.
